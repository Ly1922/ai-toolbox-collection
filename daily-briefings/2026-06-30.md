## 🔴 高信号（重点关注）

### 1. Ornith-1.0 — 自进化开源 Agentic Coding 模型族
- **来源**：Hacker News（149 pts + 54 pts）+ GitHub
- **链接**：https://github.com/deepreinforce-ai/Ornith-1
- **摘要**：DeepReinforce 发布 Ornith-1.0，一个自改进（self-scaffolding）的**开源** Agentic Coding 模型族，覆盖 **9B Dense → 397B MoE** 全尺度。专为代码代理任务设计，在 SWE-Bench 等 benchmark 上表现亮眼。9B 小模型可跑边缘设备，397B MoE 对标前沿。
- **为什么重要**：少有的聚焦「编码 Agent 专用模型」的开源方案，覆盖从端侧到数据中心全场景。
- **行动建议**：关注其 9B/31B 模型在本地 llama.cpp 上的运行表现，后续可与 Qwen 3.6 27B 做对比评测。

### 2. vLLM Micro-Agent — 在模型 API 内部协作超越前沿模型
- **来源**：Hacker News（50 pts）+ vLLM Blog
- **链接**：https://vllm.ai/blog/2026-06-29-micro-agent-frontier-models
- **摘要**：vLLM 团队发布「Micro-Agent」概念——通过 vLLM Semantic Router 将多个小模型编排为一个 bounded micro-agent runtime，利用 Confidence Routing、Ratings、ReMoM、Fusion 等策略，让多个小模型协作超越单个前沿大模型。
- **为什么重要**：直接在推理层做 Agentic 路由，无需外部编排层。对降低 API 成本、提升推理吞吐有直接影响。
- **行动建议**：实验将 Hermes 的 provider 路由与此结合，用小模型处理简单查询，大模型只处理高难度 case。

### 3. Headroom — Token 压缩工具（60-95% 缩减）
- **来源**：GitHub（⭐53,915）
- **链接**：https://github.com/headroomlabs-ai/headroom
- **摘要**：在 token 到达 LLM 之前，对工具输出、日志、文件、RAG chunks 进行压缩，号称**减少 60-95% 的 token 消耗**，输出质量不变。支持库、代理、MCP 服务器三种部署方式。
- **为什么重要**：Token 成本是 Agent 长期运行的瓶颈。如果压缩率真实，能直接降低运营成本 50%+。
- **行动建议**：作为 MCP server 集成测试，对比压缩前后的 Agent 任务质量。

### 5. Open Memory Protocol — AI 记忆的开放标准
- **来源**：GitHub（⭐6，近日新项目）
- **链接**：https://github.com/SMJAI/open-memory-protocol
- **摘要**：一种用于 AI 记忆的可移植、可互操作的开放标准协议，支持跨工具（Claude、ChatGPT、Cursor 等）、跨会话、跨设备共享记忆。采用 MCP 协议，支持自托管。
